本週 AI 動態：三份摘要拆給你看

整合日 · 時事節

🎯 今夜主題

今晚不上新課、換個角色：把過去一週信箱裡躺著的三封 Perplexity 產業摘要拆開讀。原則很簡單——摘要本身是二手整理，凡是能查到官方一手來源的，附上連結；查不到的，老實說「這是摘要的說法，未逐條查證」。

①

這週最大背景：模型與代理集中發布，但比拼的東西變了

本週的產業摘要指出：多家公司在同一週密集推出新一代模型與可持續運行的背景代理（background agent），競爭重點已經從「參數規模」轉向「推理效率、成本、以及 agent 之間能不能互通（agent interoperability）」——換句話說，重點不再是單次問答有多聰明，而是能不能撐起一整條多步驟、跨裝置的工作流程。

這一條是 Perplexity 摘要的整體判讀，本身即為二手綜合意見，未逐一查證每家公司的一手公告。

來源：Perplexity 每週摘要《本週全球 AI 與新興技術格局》（2026-07-10，team@mail.perplexity.ai）

②

兩件查證屬實、跟你工作流有關的更新

Claude Science（2026-06-30 beta 上線）：Anthropic 推出的科研工作台，重點不是新模型、而是工作流——每一個產出的圖表都附上生成它的原始碼、環境設定與完整訊息紀錄，設計成「協調 agent 負責分派、獨立的審查 agent 負責檢查引用文獻、數字與圖表」的雙代理架構，目的是讓 AI 產出的科研結果可被追溯、可重現。Claude Fable 5 / Mythos 5（2026-06-09 上線）：兩者是同一底層模型的不同安全分級版本——Fable 5 已全面開放（API／Bedrock／Google Cloud／Microsoft Foundry 皆可用），Mythos 5 因具備較強的生物與資安能力、目前僅開放給少數經審核的合作夥伴（Project Glasswing）。

來源：TechCrunch｜Anthropic's Claude Science bets on workflow, not a new model（2026-06-30）、Claude Platform Docs｜Introducing Claude Fable 5 and Claude Mythos 5

③

一個摘要講得不夠精確的地方：Agents SDK 不是「這週」的新聞

摘要把 OpenAI 的 Responses API 和 Agents SDK 跟「近期變動」放在一起講，容易讓人以為是這一兩週剛發生的事——查證後發現，這其實是 2025 年 3 月 11 日就發布的框架，如今已經是業界搭建多步驟 agent 工作流程的標準配備之一，並非新消息。原始 README 裡的定義很直白：「一個輕量卻強大、用來搭建多代理工作流程的框架」。放在整合日提這一條，是提醒自己：讀二手摘要時，時間感常常會被「近期」這種模糊字眼壓縮或拉長，查一次原始發布日期，通常幾分鐘就能還原真相。

來源：GitHub｜openai/openai-agents-python README

④

本週摘要給臨床工作者的建議：工具分工，而不是找一個全能工具

本週摘要的判讀是：截至目前沒有明確針對臨床或科研流程的新功能上線，所以重點該放在善用既有工具的分工，而不是追新——ChatGPT 適合撰寫初稿與臨床文件、Claude 擅長長文件理解與批判性閱讀、Perplexity 適合需要附引用的查證工作、Gemini／Copilot 則適合處理行政文件與簡報整理。摘要建議的具體做法是：用一個有引用來源的工具做實證查證，再固定用一個主力 LLM 設計格式化的工作流程，藉此壓縮寫作與行政時間、同時保留臨床判斷的主控權。

這一條為摘要的建議性判讀，屬於工具使用心法而非可逐條查證的技術規格，故未附加一手來源。

來源：Perplexity 每週摘要《本週醫療與科研 AI 工具動向》（2026-07-06，team@mail.perplexity.ai）
